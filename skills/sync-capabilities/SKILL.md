# Sync Capabilities Manifest

Inventory all Claude Code capabilities (MCP servers, installed skills, CLI tools) and
write a manifest file that Claude Desktop/CoWork can reference.

---

## Purpose

Claude Code (the CLI) and Claude Desktop/CoWork are completely siloed environments with
no awareness of each other. This skill bridges the gap by generating a capabilities
manifest — a structured file that describes everything Claude Code has access to — which
CoWork can read to understand what the CLI can do and when to suggest the user switch to it.

---

## When This Skill Activates

- "Sync capabilities"
- "Update capabilities manifest"
- "Refresh what CoWork knows about Claude Code"
- After installing a new MCP server, skill, or CLI tool (suggest running this)

---

## Workflow

### Step 1 — Inventory MCP Servers

Run:

```bash
claude mcp list 2>/dev/null
```

Parse the output to extract server names, endpoints, and connection status.

### Step 2 — Inventory Installed Skills

List all skill directories:

```bash
ls ~/.claude/skills/
```

For each skill, read the YAML frontmatter from `SKILL.md` to extract the skill name and
one-line description.

### Step 3 — Inventory CLI Tools

Check availability of key CLI tools:

```bash
which notebooklm yt-dlp gh node npm python3 playwright 2>/dev/null
```

For each available tool, note the path and version if easily obtainable.

### Step 4 — Write the Manifest

Write the complete manifest to:

```
~/Development/claude-capabilities.md
```

Use the following format:

```markdown
# Claude Code Capabilities

> Auto-generated by `sync-capabilities` skill. Last updated: YYYY-MM-DD HH:MM
>
> This file is read by Claude Desktop/CoWork to understand what Claude Code (the CLI)
> has access to. If a task requires tools listed here that CoWork cannot access directly,
> suggest the user run it in Claude Code instead.

## MCP Servers (Remote Integrations)

| Server | Endpoint | Status |
|--------|----------|--------|
| ...    | ...      | ...    |

## Installed Skills

| Skill | Trigger | Description |
|-------|---------|-------------|
| ...   | ...     | ...         |

## CLI Tools

| Tool | Path | Version | Purpose |
|------|------|---------|---------|
| ...  | ...  | ...     | ...     |

## What Claude Code Can Do

[Brief summary of capabilities based on the inventory above, written for CoWork's
consumption — e.g., "Claude Code can search YouTube via yt-dlp, manage NotebookLM
notebooks, interact with GitHub repos via gh CLI, resolve LinkedIn URLs, and access
Slack/Notion/Gmail/Calendar via MCP servers."]

### When to Hand Off to Claude Code

[List of task types that should be routed to Claude Code rather than handled by CoWork]
```

### Step 5 — Confirm

Report what changed since the last manifest (if a previous version exists) and confirm
the file was written.

---

## CoWork Integration

For CoWork to discover the manifest, place a `CLAUDE.md` in a parent directory (e.g.,
`~/Development/CLAUDE.md`) that instructs CoWork to read `claude-capabilities.md` when
it needs to understand Claude Code's capabilities.

CoWork will see this `CLAUDE.md` whenever it is scoped to any folder under that directory.

---

## Error Handling

| Error | Cause | Action |
|---|---|---|
| `claude mcp list` fails | CLI not configured | Note that MCP section may be incomplete |
| Skill directory missing | No skills installed | Report empty skills section |
| CLI tool not found | Not installed | Omit from manifest |
